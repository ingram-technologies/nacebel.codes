# Product

## Register

brand

## Users

Belgian founders, accountants, bookkeepers, lawyers and civil servants — plus
anyone integrating the data via the API. Their context is almost always mid-task:
registering or amending a company at the KBO/BCE, filing VAT, checking a tender's
eligibility, or wiring the classification into their own software. They arrive
knowing roughly what an activity is and need the *exact* NACE-BEL 2025 code, fast,
with enough surrounding structure (parent, children, level) to be sure it's right.

## Product Purpose

nacebel.codes is a free, fast, well-typeset reference for the **NACE-BEL 2025**
classification of economic activities in Belgium — a searchable directory of all
~3,255 codes, one canonical page per code (parent, children, level, official KBO
link), a CSV export, and a public API. Success = the visitor finds and trusts the
right code in seconds and can cite or integrate it without second-guessing. It is
an index and a utility, not a campaign.

## Brand Personality

Authoritative, precise, quietly confident. The voice of a well-made official
reference — a statistical yearbook or a good API doc — not a startup landing page.
Three words: **exact, legible, unhurried.** It should feel trustworthy enough to
cite and calm enough to read for a while.

## Anti-references

- Glassmorphism SaaS landing pages: frosted white cards, blurred aurora-gradient
  backgrounds, big soft shadows. (The current site leans this way — move away.)
- The warm cream/sand "editorial-warm" body background that every AI generates.
- Rainbow-coded UI where color is decoration, not information.
- Tiny uppercase wide-tracked eyebrows stacked above every section.
- Government-portal drabness: grey Arial on white, no craft, no hierarchy.

## Design Principles

- **The code is the hero.** Monospace is a first-class type role; codes are set,
  aligned, and copyable everywhere. The interface serves scanning and certainty.
- **Density with calm.** This is reference data — favour a tight, hairline-ruled
  index over generous floating cards. Rhythm and rules carry structure, not shadow.
- **Structure is legible.** The 5-level hierarchy (section → class) is always
  visible and encoded consistently (depth, not a random palette), so users can
  place any code in the tree at a glance.
- **Earn trust through craft and restraint.** Authority comes from typography,
  contrast and precision, not decoration. One disciplined accent; no effect for
  effect's sake.
- **Practical first.** Search, copy, export, deep-link, and the API are the point;
  every visual choice must make those faster, never slower.

## Accessibility & Inclusion

Target WCAG 2.1 AA. Body and placeholder text ≥4.5:1, large text ≥3:1 — no light
grey body text. Full light and dark themes. Level colours must stay distinguishable
for colour-blind users (carry depth via label/shape, not hue alone). All animation
respects `prefers-reduced-motion`. Keyboard-navigable search, results, and menus.
